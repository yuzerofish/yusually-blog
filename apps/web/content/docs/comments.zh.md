## 评论系统是什么

评论系统让读者可以**在你的每篇文章下面留言**，就像微信公众号的留言区一样。读者看完文章后，可以发表想法、提问或者给你反馈。

## 为什么需要登录才能评论

和知乎、微博一样，读者需要先登录才能发表评论。这样做的好处是：

| 好处       | 说明                     |
| ---------- | ------------------------ |
| 防垃圾评论 | 机器人没法随便刷评论     |
| 可追溯     | 每条评论都有明确的发布者 |
| 管理方便   | 可以针对具体用户进行管理 |

## 读者的登录方式

你的读者有三种方式登录后发表评论：

- **GitHub 一键登录** — 点一下就能用 GitHub 账号登录（程序员读者最方便）
- **Google 一键登录** — 点一下就能用 Google 账号登录（大多数人都有）
- **邮箱 + 密码** — 用邮箱注册一个账号登录（备用方式，任何人都能用）

> GitHub 和 Google 登录需要技术人员帮你配置。如果暂时没配置，读者可以先用邮箱密码方式注册评论。

## 评论审核机制

你可以选择两种审核模式：

| 模式         | 效果                                       | 适合场景               |
| ------------ | ------------------------------------------ | ---------------------- |
| **先审后发** | 评论提交后不会立即显示，等你审核通过才公开 | 对内容要求严格的站点   |
| **直接发布** | 评论提交后立即公开显示                     | 信任读者、追求互动效率 |

在后台设置页面，一键切换即可，不需要改代码或重新部署。

## 关键词屏蔽

你可以设置一组**屏蔽关键词**（比如广告词、脏话等）。当读者的评论包含这些词时：

- **开启自动屏蔽**：评论会被自动标记为垃圾评论，不会公开显示
- **关闭自动屏蔽**：评论会进入审核队列，由你手动判断是否通过

关键词列表可以随时在后台修改，不需要技术人员帮忙。

## 管理员怎么操作

所有评论管理都在你的**后台管理面板**完成：

- 进入 **评论管理页面**（`/admin/comments`）可以：审核通过、标记为垃圾评论、或删除评论
- 进入 **设置页面**（`/admin/settings`）可以：开关评论功能、切换审核模式、管理屏蔽关键词

> 这些操作都在网页后台点击完成，不需要写代码，也不需要重新部署网站。
